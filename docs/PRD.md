# 车辆保养记录App - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
一款简单易用的车辆保养记录工具，帮助用户管理和追踪车辆保养历史，数据完全存储在本地，无需联网。

### 1.2 目标用户
- 私家车车主
- 需要记录车辆保养信息的用户
- 关注车辆维护和保养的用户

### 1.3 核心价值
- 便捷记录每次保养信息
- 自动计算保养间隔
- 本地数据存储，保护隐私
- 跨平台支持（iOS/Android）

---

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 保养记录管理
- **添加保养记录**
  - 保养日期（日期选择器）
  - 保养时里程（数字输入）
  - 保养项目（多选/自定义输入）
  - 保养价格（金额输入）
  - 备注（文本输入）
  
- **查看保养记录列表**
  - 按时间倒序排列
  - 显示关键信息（日期、里程、价格）
  - 显示与上次保养的时间差、里程差
  
- **编辑保养记录**
  - 修改已有记录的所有字段
  
- **删除保养记录**
  - 删除单条记录
  - 确认提示

#### 2.1.2 数据统计
- **保养统计概览**
  - 总保养次数
  - 总保养费用
  - 平均保养间隔（时间）
  - 平均保养间隔（里程）
  
- **保养项目统计**
  - 各保养项目的次数统计
  - 各保养项目的费用统计

#### 2.1.3 数据管理
- **数据导出**
  - 导出为CSV格式
  - 导出为JSON格式
  
- **数据导入**
  - 从CSV/JSON文件导入数据
  
- **数据备份**
  - 手动备份数据库文件

### 2.2 次要功能

#### 2.2.1 车辆信息管理
- 支持添加多辆车
- 每辆车独立的保养记录
- 车辆基本信息（品牌、型号、车牌号、购买日期等）

#### 2.2.2 保养提醒
- 基于时间的提醒（如距离上次保养6个月）
- 基于里程的提醒（如距离上次保养5000公里）
- 推送通知提醒

## 3. 数据模型

### 3.1 车辆信息 (Vehicle)
```
{
  id: String (UUID)
  name: String (车辆名称)
  brand: String (品牌)
  model: String (型号)
  licensePlate: String? (车牌号)
  purchaseDate: DateTime? (购买日期)
  initialMileage: double? (初始里程)
  createdAt: DateTime
  updatedAt: DateTime
}
```

### 3.2 保养记录 (MaintenanceRecord)
```
{
  id: String (UUID)
  vehicleId: String (关联车辆ID)
  date: DateTime (保养日期)
  mileage: double (保养时里程)
  items: List<String> (保养项目)
  price: double (保养价格)
  notes: String? (备注)
  timeDiffFromLast: Duration? (与上次保养时间差)
  mileageDiffFromLast: double? (与上次保养里程差)
  createdAt: DateTime
  updatedAt: DateTime
}
```

### 3.3 保养项目 (MaintenanceItem)
```
{
  id: String (UUID)
  name: String (项目名称)
  isDefault: boolean (是否为默认项目)
  sortOrder: int (排序)
}
```

## 4. 用户界面设计

### 4.1 页面结构
```
├── 首页/记录列表页
├── 添加/编辑记录页
├── 统计页
├── 车辆管理页
├── 设置页
└── 数据导出/导入页
```

### 4.2 设计原则
- 简洁明了，操作便捷
- 符合Material Design设计规范
- 支持深色模式
- 响应式布局，适配不同屏幕尺寸

---

## 5. 非功能性需求

### 5.1 性能要求
- 应用启动时间 < 2秒
- 列表滚动流畅，无明显卡顿
- 数据库操作响应及时

### 5.2 数据安全
- 数据完全本地存储，不上传服务器
- 支持数据加密（可选增强功能）

### 5.3 兼容性
- iOS 12.0+
- Android 6.0 (API 23)+
- 手机和平板适配

### 5.4 可靠性
- 数据操作异常处理
- 数据库版本升级兼容
- 数据备份恢复机制
